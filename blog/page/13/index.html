Leo给程序员的09年建议(zz)

2008-12-31

via: http://blog.csdn.net/jobchanceleo/archive/2008/12/31/3662263.aspx 职场立身经

让自己有所长，才是职场立身根本

最近，听裁员这个词儿的次数比听饭岛爱还 多，身边一些朋友也受到了直接的影响，元旦前回空休息了。有些朋友问：“怎么才能不被裁？”其实，没有什么一定之规，但我知道一种方法可以大大减少自己被 “砍”的风险，那就是会一样很少人会，公司开发中又用的到的开发技术或者工具。我亲眼见过一个技术平平、薪水很高的哥们，在历欠裁员中被保留，只因为他掌 握一项技术，公司用的上，会的人又很少。

其实，不用那么消极的考虑如何不被裁，应该问的是：“会什么技术我才能走到哪吃到哪？”我的回答同上。

心态定胜负

事事岂能尽如人意，但求无亏于心

很 多时候，在工作中我们已竭尽全力，但结果并不尽如人意。那么请首先检讨一下自己，问问自己：“我是否真的尽力了？”如果回答“是”。即使结果不那么令人满 意，从个人的角度上讲也完全可以接受，一是可能项目受客观因素的影响，比如：客观条件、资源情况等，二是既然尽力，还没达到预期，说明我们还有提高的余 地，还有继续学习的空间（是件好事情）。

总之，尽力就好，很多时候，结果如何要看缘分。无论发生什么，明天太阳依然升起。只要我们能做到持续努力，不断学习就好J

Slackware 12.2的vim Bug

2008-12-26

在vim里，用shift + k来快速调用man，对于coder来说是非常常用的，但在最新slackware版本里有些问题 其实是vim的bug，只是slackware没把其较新的补丁合入，做法: 
wget -r ftp://ftp.vim.org/pub/vim/patches/7.2/
cd ftp.vim.org/pub/vim/patches/7.2/
rm MD5 MD5SUMS README
gzip *    # 这是为了符合SlackBuild打patch的做法
cp *.gz source/ap/vim/patches/
*.gz是刚才gz的补丁文件, 目标目录是SlackBuild目录 ./SlackBuild，生成新包在/tmp下，upgradepkg即可 可参考： http://linuxsir.org/bbs/showthread.php?p=1930015#post1930015

slackware12.1简单升级到12.2

2008-12-18

今晚12.2的DVD ISO下载完，使用slackpkg进行了简单快捷的升级，下面是我总结的一些方法，希望对大家有用： 1、首得建议telinit 1，进入单用户模式先，这个防止升级glibc时出问题，接着当然得mount上这个ISO啦，记得要-o loop参数 2、系统里要先有slackpkg，这个东东已合到12.2的ap分支里，已经官方支持，建议大家使用这个代替slapt-get; 3、安装完slackpkg，修改/etc/slackpkg/mirrors，把cdrom那一行激活，并修改为ISO的那个目录，我挂到/mnt，所以改为cdrom://mnt就好 4、接下来的过程和平时用slackpkg一样，先slackpkg update一下，再slackpkg upgrade-all，选取自己想升的包，OK 5、升完后，会提示是否要lilo一下，这个是必须的，不然启动时会kernel panic的哦，如果用grub的话，需要自己改一下menu.lst 6、之后会有新配置文件，提示你操作，我建议为选K，就是新配置文件改名为.new，这样大家可以一个个diff一下，再mv就好啦，不然有些配置得重新配，我个人喜欢这样 7、最后，如果你不放心，重启前再lilo一下，reboot重启，一切OK，简单吧？！ 8、选下载ISO的好处是，下载时快，升级也快，也可以备份一下，里面的source目录方便后面的SlackBuild 这个两个版本更新的东东不少，内核从2.6.24升到2.6.27，用hdparm -tT /dev/sda的结果去到116M/s啦，之前一直只有一半的速度:-) 这是自己总结的方法，如有不足的地方，欢迎交流LOL enjoy~

关于proftpd开启anonymous访问权限

2008-12-15

proftpd是Linux下很常用的一个Ftp服务器，slackware自带vsftpd和proftpd两个ftp服务器。 vsftpd小巧快速，proftpd在定制性和安全性比vsftpd好。 我打算把proftpd开anonymous（匿名）访问权限，slackware 12.2安装时，proftpd已把/etc/proftpd.conf配置好支持anonymous的，但登录时老出现: 220 ProFTPD 1.3.1 Server (ProFTPD Default Installation) [::ffff:127.0.0.1] Name (localhost:root): anonymous 331 Anonymous login ok, send your complete email address as your password Password: 530 Login incorrect. Login failed. 从而失败。上网查了很多资料，但没找到解决方法。后来发现是自己没有认真阅读proftpd.conf，人家里面就有一句: 
# To enable this, remove the user ftp from /etc/ftpusers.
打开/etc/ftpusers，人家里面又有一句: 
# To enable anonymous FTP, remove the "ftp" user:
我晕，那只要在ftp前加个#后，
/etc/rc.d/rc.inetd restart
就行啦。 总结一下： 1、我想不明白为什么网上没有这个帮助资料，可能我比较菜，也比较粗心； 2、网上提到开anonymous高频help是加入
RequireValidShell off
slackware默认安装已有这个参数； 3、借此分享给大家

关于slackware的字体美化

2008-12-13

Linux的美化可谓是个永恒的话题，我记得以前还为美化这个东东搞过几天，还没达到自己的要求（相对XP）。 可是呢，人靠衣服，系统也希望弄得好看点，这样心情才好呢，这次来说说slackware的字体美化，也算是Linux的通用方法。 有两个链接可以参考的: http://linuxsir.org/bbs/thread339680.html http://www.linuxquestions.org/questions/slackware-14/beautiful-fonts-in-12.1-640468/ 后面那个链接还把libXft和cairo换成LCD适应的，有这个就更好，该网站也提供tgz包下载，直接installpkg就好了，非常方便。但我个人觉得把FreeType的选项打开就好，方法有两个： 1、用slackware的有福，pat已在SlackBuild里准备好，可以从source/l/freetype那里找到，把两个zcat前面的#去掉，这里也可以顺便将其编译为i686架构的，然后./SlackBuild一下，生成新的tgz在/tmp目录下，upgradepkg之即可，爽吧？ 2、geeker们可以自己去改FreeType的头文件，这个可以google一下，也就是手动打开宏定义而已 另slackware12.1只带wqy-zenhei是不够的，可以再加入wqy-bitmapfont（个人喜欢其对细字体的优秀支持），INSTALL说明一步步装就好了，上面第一个链接也有这个说明。 另个人推荐把startx的dpi设为96，可以startx -dpi 96或编编辑/usr/bin/startx，改为： sysserverrc=/usr/lib/X11/xinit/xserverrc defaultclient=xterm defaultserver=/usr/bin/X defaultclientargs=”” defaultserverargs=”-dpi 96” clientargs=”” serverargs=”” （来自第二个链接） 至此完成美化了，不同发行版方行都是差不多的。 有些人是看不到漂亮字体就不爽的，估计要花些时间配啦，什么雅黑啊，苹果的字体都可以用上，我以前看过一些图很不错的，想弄可以google之。
